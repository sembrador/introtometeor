How does it work? (step3)

Meteor.Collection (/lib/Bands.js)
  Bands = new Meteor.Collection('bands')

Server publishes with overlap. (/server/bands.js)
  Meteor.publish("allbands", function(arg1,...) {...})

Client subscribes, optionally reactive. (/client/subscribe.js)
  Meteor.subscribe('allbands', arg1,...)

Server publishes docs or changes to all client subscriptions and patches up dom.
Meteor.Collection - connection, idGeneration, transform (fetch, findOne, observe, allow, deny)

Meteor.Collection (step3)

find, findOne(selector, [options])
  selector: mongo query or string {name:'13th Floor Elevators'}
  [options]: sort, skip, fields, reactive, tranform

insert(doc, [cb(error, result)]) on server no cb blocks returns id or throws e

update(selector, modifier, [options{multi:bool}], [cb(error)]) client by _id only, server no cb blocks or throws e.

remove(selector, [cb(error)]) client by _id only, server no cb blocks or throws e.

allow and deny - more on that later!

Lets get CRUDy (step3)

Read relies on subscriptions and client side queries.

Client side
  Bands.insert({name: 'Motorhead', votes: 999999999})
  Bands.update({_id: id}, {$set:{votes: 4}})
  Bands.remove({_id: id})
* can only update or remove by _id

Server side. Anything goes!
  Bands.update({name: 'Back Flag'}, {$set:{name:'Black Flag'}})
  Bands.remove({votes: {$lte: 0}}, {multi:true})
* multi: true required to remove multiple records

Allow Deny (step3)

Callbacks to restrict insert, update and remove from client calls.

If one Allow is true then action occurs.

Except that Deny overrides ALL allow rules.

fetch - string array to define what fields are available.

transform - can override the Meteor.Collection transform.

Bands.insert({name: '13th Floor Elevators', votes: 98989}) from client could fail!

observe (step4)

Watch a live query (cursor) and get results as it changes.
  added(document)
  addedAt(document, atIndex, before)
  changed(newDocument, oldDocument)
  changedAt(newDocument, oldDocument, atIndex)   removed(oldDocument)
  removedAt(oldDocument, atIndex)

observeChanges

Only differences of old and new docs passed.
  added(id, fields)
  addedBefore(id, fields, before)
  changed(id, fields)
  movedBefore(id, before)
  removed(id)
